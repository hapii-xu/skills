# Workflow: 后端调试

## 触发条件
用户说：接口报错、500错误、数据库异常、性能问题、超时

## 执行步骤

### Step 1: 复现与定位
1. 获取错误信息（状态码、错误消息、堆栈）
2. 确认请求参数（method, url, body, headers）
3. 定位出错文件（路由？Service？数据库？）

### Step 2: 按错误类型排查

**500 Internal Server Error**：
```
1. 查看错误堆栈
2. 检查是否是未捕获异常
3. 检查数据库连接
4. 检查外部服务调用
```

**数据库相关错误**：
```
SQLITE_CONSTRAINT → 唯一约束冲突 / 外键冲突
SQLITE_BUSY       → 数据库锁 → 检查长事务
ENOENT            → 数据库文件不存在 → 检查路径
```

**认证/权限错误**：
```
401 → Token 过期或无效 → 检查 JWT 验证逻辑
403 → 权限不足 → 检查角色/权限配置
```

### Step 3: 修复
- 确认根因（不要只修表面症状）
- 最小改动修复
- 添加错误处理防止复发

### Step 4: 验证
- 原问题已修复
- 关联功能不受影响
- 添加回归测试

## 常见错误速查

| 错误 | 常见原因 | 排查方向 |
|------|---------|---------|
| 500 | 未捕获异常 | 检查 try-catch、null 检查 |
| 400 | 参数验证失败 | 检查 schema 定义 |
| 404 | 路由不匹配 | 检查 URL 拼写、路由注册 |
| 409 | 唯一约束冲突 | 检查重复数据 |
| 超时 | 慢查询/死锁 | 检查 SQL 性能、连接池 |

## 输出格式
```markdown
### Bug 修复：[接口/问题描述]

**根因**：[一句话说明]
**修复**：[修改了什么]
**预防措施**：[如何防止复发]
```
