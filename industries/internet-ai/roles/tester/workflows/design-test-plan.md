# Workflow: 设计测试计划

## 触发条件
用户说：测试计划、测试策略、测试范围、质量保障

## 执行步骤

### Step 1: 确定测试范围
1. 阅读 PRD，识别所有功能模块
2. 标记每个模块的风险等级（高/中/低）
3. 确定测试优先级：高风险模块优先

### Step 2: 选择测试策略
| 功能类型 | 测试方法 | 工具 |
|---------|---------|------|
| 工具函数 | 单元测试 | Vitest |
| API 端点 | 集成测试 | Supertest + Vitest |
| 用户流程 | E2E 测试 | Playwright |
| UI 组件 | 组件测试 | Vitest + Testing Library |

### Step 3: 输出测试计划
```markdown
## 测试计划：[功能名称]

### 测试范围
- [模块1]：核心功能，需完整测试
- [模块2]：辅助功能，需基本测试
- [模块3]：低风险，仅冒烟测试

### 测试策略
| 层级 | 覆盖范围 | 工具 | 目标 |
|------|---------|------|------|
| 单元测试 | Service/Utils | Vitest | >80% |
| 集成测试 | API 端点 | Supertest | 核心流程100% |
| E2E | 关键用户路径 | Playwright | P0功能100% |

### 测试环境
- 测试数据库：SQLite 内存数据库
- 测试数据：种子脚本提供
- CI 集成：GitHub Actions

### 里程碑
1. 核心模块测试用例 → [时间]
2. 集成测试完成 → [时间]
3. E2E 回归测试 → [时间]
```
