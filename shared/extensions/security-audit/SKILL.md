# 安全审计技能

## OWASP Top 10 检查项

### 注入攻击
- SQL 注入：使用参数化查询，禁止拼接 SQL
- 命令注入：避免直接执行用户输入的 shell 命令
- XSS：输出编码，使用 CSP 策略

### 身份认证
- 密码使用 bcrypt/argon2 哈希存储
- Session/Token 有过期机制
- 登录失败有频率限制
- 支持多因素认证（MFA）

### 访问控制
- 最小权限原则
- API 接口级别权限校验
- 避免水平越权和垂直越权
- 管理接口与普通接口隔离

### 数据保护
- 敏感数据传输使用 HTTPS
- 敏感数据存储加密
- 日志中不记录密码、Token 等敏感信息
- 遵守数据保护法规

### 安全配置
- 关闭不必要的端口和服务
- 安全 HTTP Headers（CSP, X-Frame-Options, HSTS）
- 错误信息不暴露内部实现
- 依赖包定期更新，修复已知漏洞

### 审计流程
1. 识别所有输入点和数据流
2. 检查每个输入的验证和清理
3. 检查认证和授权逻辑
4. 检查数据存储安全
5. 检查第三方依赖安全
6. 生成审计报告和修复建议

### 安全开发原则
- 永远不信任用户输入
- 纵深防御（多层安全措施）
- 安全默认（默认拒绝）
- 失败安全（出错时偏向更安全的状态）
