# Code Review 技能

## 审查清单

### 代码质量
- [ ] 命名是否清晰表达意图
- [ ] 函数/方法是否单一职责
- [ ] 是否有重复代码可提取
- [ ] 错误处理是否完善
- [ ] 类型定义是否完整

### 安全检查
- [ ] 输入验证是否到位
- [ ] SQL 查询是否使用参数化
- [ ] 是否存在 XSS/注入风险
- [ ] 敏感数据是否加密
- [ ] 权限校验是否完整

### 性能检查
- [ ] 是否有不必要的重复计算
- [ ] 数据库查询是否高效（索引、N+1）
- [ ] 大列表是否使用虚拟滚动/分页
- [ ] 资源是否正确释放

### 可维护性
- [ ] 注释是否解释"为什么"而非"做什么"
- [ ] 复杂逻辑是否有文档说明
- [ ] 是否遵循项目代码风格
- [ ] 测试是否覆盖关键路径

## 审查原则
1. 对事不对人，关注代码质量
2. 提供具体的改进建议，不只说"不好"
3. 区分"必须改"和"建议改"
4. 认可写得好的代码
